# `patches/_archive_superseded/` — DO NOT APPLY

These three patches are **superseded** by newer code that handles their
concerns better. Applying any of them would be wrong.

## What's in here

### `C01i_cheat24_merge_patch.R`

**Was:** a tiny merge that puts `cheat24`'s `event_class` onto the main
C01i `result` data.table, so the old `C01i_recombinant_groups_patch.R`
could subset RECOMBINANT_GC / RECOMBINANT_DCO subgroups.

**Why superseded:** `4b_group_proposal/STEP_C01i_b_multi_recomb.R`
(Phase 4b.2 in the new architecture) runs `cheat24` classification
end-to-end as part of its three-signal recombinant detection. The
`event_class` is produced inside the rewrite, written directly to
`recombinant_map.json`, and consumed by `STEP_C01i_d_seal.R` (4b.4).
No merge step needed.

**Explicit confirmation from v10.1.1 ADDITIONS doc:**

> Cheat24 event_class merge into C01i_decompose's `result` table is no
> longer needed — the new `STEP_C01i_b_multi_recomb.R` handles cheat24
> classification end-to-end. The v10 patch `C01i_cheat24_merge_patch.R`
> can be skipped.

### `C01i_recombinant_groups_patch.R`

**Was:** an in-place edit to the OLD `STEP_C01i_decomposition_rewired_24.R`
that replaced the 3-class registration loop (HOM_STD/HET/HOM_INV) with
a 4-class block that added RECOMBINANT plus optional GC/DCO subgroups.

**Why superseded:** the entire old `STEP_C01i_decomposition_rewired_24.R`
has been replaced by the four-script subsystem in `4b_group_proposal/`.
The new `STEP_C01i_d_seal.R` (4b.4) registers all four groups (HOM_REF,
HET, HOM_INV, RECOMBINANT) + the optional GC/DCO subgroups as a core
responsibility, not as a retrofit patch. It does this better because:

- Proper synthesis across all three upstream blocks (decompose,
  multi_recomb, nested_composition) rather than just splitting by
  `event_class`
- Writes the composite-flag promotion cap alongside the group
  registration
- Sets `q6_group_validation = UNCERTAIN` with `q6_validation_promotion_cap`
  correctly, which the old patch could not

**Explicit confirmation from phase4b_rewrite/README.md:**

> This is additive on top of v10. It does not delete or conflict with
> v10's existing files — it **supersedes
> `patches/C01i/C01i_recombinant_groups_patch.R`** because the new
> `STEP_C01i_d_seal.R` registers groups properly.

## What happens if you apply them anyway

- `C01i_cheat24_merge_patch.R`: harmless but redundant. The merge has no
  effect because the new code doesn't read the merged column.
- `C01i_recombinant_groups_patch.R`: will fail because the script it
  patches (`STEP_C01i_decomposition_rewired_24_v934_registry.R`) is no
  longer the active script. You'd need the legacy file to apply it to.

## Kept for reference

These are preserved because the design logic inside them (how GC vs DCO
recombinants were originally identified, the 4-class registration
semantics) was the blueprint for the rewrite. Useful as a reference if
you ever need to trace a decision back to its origin. Not useful as
runnable code.

### `patch_C01f_hypothesis_flashlight.R`

**Was:** a legacy integration patch that added "T9: SV GENOTYPE CONCORDANCE"
to C01f — Fisher-exact test on a 2×3 contingency table of DELLY/Manta
SV genotype × PCA band, plus a Cheat 2 sub-test for het-DELs at
breakpoints.

**Why superseded:** three reasons, any one of which is disqualifying:

1. **Slot collision.** In v9.3.2+ C01f, the T9 slot is now occupied by
   the ancestry jackknife (Cheat 6), which feeds the group-validation
   promotion logic. SV concordance signal has moved earlier in the
   pipeline — it is computed by `STEP_C00_build_sv_prior.R` in phase 2c
   and stamped onto per-window columns by `STEP_C01a_precompute.R`. By
   the time C01f runs, SV concordance has already been integrated into
   the candidate score. Re-computing it as a T9 would be double-counting.

2. **Autogen R parse bug.** Lines 33 and 35 contain R statements inside
   a `c(...)` vector constructor:

   ```r
   c(
     file.path(dirname(outdir), "utils", "flashlight_loader_v2.R")
     if (!file.exists(fl_loader_path)) fl_loader_path <- sub("_v2", "", fl_loader_path),
     ...
   )
   ```

   This is not valid R; the `if (...) ...` statement must live outside
   the `c()` call. It was never a working patch — it was generated by
   an earlier LLM-assisted pass that did not run through R's parser.

3. **Flashlight terminology retired.** The pipeline has migrated
   `flashlight` → `sv_prior`; the patch still references
   `FLASHLIGHT_LOADER` and `flashlight_loader_v2.R`. Even if the
   autogen bug were fixed, the loader paths no longer exist.

**What replaces it:** the `sv_prior` annotation in phase 2c. Per-window
columns `sv_inv_overlap`, `sv_inv_confidence`, `sv_inv_af`,
`sv_n_anchors` are stamped on by C01a and flow into the candidate
scoring in C01d. T8 (Clair3 genotype concordance, Cheat 9) provides
the independent-genotype-vs-PCA-band concordance test that the old
flashlight T9 was trying to do, but using Clair3 SNP/indel genotypes
rather than DELLY/Manta SV calls.
