# `patches/` — legacy integration patches (reference only)

These are the six patch files that inserted SV-prior ("flashlight")
annotation logic into downstream scripts. They are preserved here as
reference material, not as drop-in patches.

## Do not apply these directly

All six patches contain the same autogeneration bug:

```r
file.path(dirname(outdir), "utils", "flashlight_loader_v2.R")
if (!file.exists(fl_loader_path)) fl_loader_path <- sub("_v2", "", fl_loader_path),
```

Statements were inserted inside a `c(...)` vector constructor, which is
not valid R syntax and will fail at parse time. The SV-prior loader
logic itself is fine; the scaffolding around it needs to be rewritten
as valid R when integrated.

## When you integrate one of these

1. Read the patch to understand intent.
2. Rewrite the loader-sourcing block using the clean form used in
   `STEP_C01a_precompute.R` (see lines around the `sv_prior_dir`
   argument parsing and the SV-PRIOR-annotation section).
3. Apply the rename rules from `../RENAMING.md` to all identifiers
   (`flashlight` → `sv_prior`, `cheat1/2/3/8` → `test_01/02/03/08`,
   `[flashlight]` → `[sv_prior]`, etc.).
4. Place the integrated block where the patch header says (after
   `inv_like_dt` is built, before the dosage-het upgrade, etc.).
5. Test on one chromosome before running across all.

## Clean loader-sourcing pattern

Use this shape wherever a script needs to load the SV prior:

```r
# Find sv_prior directory
sv_prior_dir <- NULL
# ... argument parsing sets sv_prior_dir from --sv_prior_dir CLI flag ...
if (is.null(sv_prior_dir)) {
  sv_prior_dir <- Sys.getenv("SV_PRIOR_DIR", "")
  if (!nzchar(sv_prior_dir)) {
    # Fallback: look next to the MDS output
    sv_prior_dir <- file.path(dirname(outdir), "sv_prior")
  }
}

# Read per-chromosome sv_prior with backward-compat fallback for old filenames
read_sv_prior <- function(chr, dir) {
  for (stem in c(sprintf("sv_prior_%s.rds", chr),
                 sprintf("sv_flashlight_%s.rds", chr),
                 sprintf("%s_flashlight.rds", chr))) {
    p <- file.path(dir, stem)
    if (file.exists(p)) return(tryCatch(readRDS(p), error = function(e) NULL))
  }
  NULL
}
```

This is what `STEP_C01a_precompute.R` already does (lines around the
SV-PRIOR-annotation block). Copy that pattern when integrating the
other five patches.

## Why the patches were autogen-broken

The six patches appear to have been generated by an earlier LLM-assisted
refactor that did not run through R's parser. They are useful as design
documents showing what the integration should do, but the scaffolding
around the actual logic is broken. Treat them as prose, not code.
