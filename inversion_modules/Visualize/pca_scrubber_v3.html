5 helpThis page.

Key metrics

ConcordFraction of samples assigned to the same K-means group in two adjacent L2s after Hungarian alignment. High = L2s are interchangeable; suggests merge.
Cramér's VEffect-size measure of association between two clusterings. N-independent. <0.10 small, 0.10–0.30 medium, >0.30 large.
p-valueSignificance of the contingency. With persistent sample structure, p ≈ 0 is expected and confirms association — read together with concord.
ConservationMean concord with left + right L2 neighbors. Low (orange) = the L2 boundary is biologically real.
CoherenceCorrelation between two-half mean PC1 vectors within the L2. Low (red, <0.60) = INCOHERENT, suggests over-merge of distinct sub-regions.
Fam purityWithin-cluster fraction from the dominant family. ~1.0 (red, FAM-LD?) = clusters are family-pure → likely family LD, not real inversion. ~0.2 = real biological signal.
SilhouetteCluster separation quality from K-means. >0.5 good, <0.45 weak.
K usedThe number of clusters used for this L2. In adaptive mode, picked per-L2 by silhouette.
σ verdictFor K≥4 L2s, classifies the source of extra bands.

σ verdicts (K ≥ 4 L2s)

TWO_INVERSIONSAll samples have low σ (q50 < 0.05). Joint karyotypes are stable → two stacked inversions.
CROSSOVER_ARTIFACTSBimodal σ distribution with small high-σ tail (2–25% of samples drift). Suggests gene conversion / double crossovers in heterozygotes.
NOISY_REGIONEveryone has high σ. Low informative power.
NAK used ≤ 3. No extra bands to explain.
UNDETERMINEDProfile doesn't cleanly fit any pattern.

K labels TSV (for C++ pop-stats engine)

The export K labels TSV button in the sidebar produces one row per (L2, sample) with columns: chrom, l2_id, l2_idx, l2_start_bp, l2_end_bp, l2_n_windows, ind, cga, family_id, k_used, k_label, k_label_fixed, silhouette, fam_purity, coherence, cluster_ok.

k_label is the adaptive-K assignment (varies per L2 if adaptive mode is on); k_label_fixed is always at state.k (default 3) for cross-L2 comparisons. Group samples by k_label_fixed for uniform pop-stats; by k_label for per-L2 natural clustering.

Pop-stat tracks (--track_<label>)

Pass any per-window scalar TSV at export time:

--track_theta theta.tsv --track_fst_K1K3 fst.tsv --track_pi pi.tsv

TSV must have a value column plus one of:
(a) chrom, start_bp, end_bp, value (range form)
(b) chrom, window_idx, value (1-based index form)
(c) chrom, center_bp, value (center form)

Each track renders as a 60px stacked panel between the Z and PCA panels on page 1.

Color codes (cluster mode)

K-means group → blue / grey / orange / green / red (lo to hi PC1). Samples colored consistently within an L2; Hungarian-aligned to focal in mini-PCAs.

Lock button (🔒 lock colors to current L2) freezes labels from a reference L2 → samples keep those colors as you scrub through other L2s. Useful for tracking how a clean K=3 split holds up across boundaries.

Note on manuscript writing

Catalogue exports (TSV / Markdown) are designed to drop into Methods / Results sections. They're aids to writing — they don't write the manuscript. Nature acceptance depends on the underlying biology, the prose, and peer review.

Roadmap — features deferred for later turns

Tracking what we want to add but didn't fit in this iteration. Each entry lists what it would need to actually build, so we don't lose the thread next time.

Per-sample H per band
inversion vs not inversion		 Compute heterozygosity for each sample inside the candidate window range, grouped by their locked band. Needs per-sample H emitted by the C++ pop-stats engine as a track (e.g. --track_h_persample). Once available, scrubber renders one bar chart per band on page 2 below the band-composition cards.
π / θ curve per band
nucleotide diversity profile		 Per-band θ along the candidate span requires the C++ engine to compute θ separately for each k-label group and emit one track per band. Then the scrubber renders these as overlaid lines on the candidate page. Could also do this on page 1 as multi-track overlay if tracks are flagged as "stratified by band".
Gene annotations
GFF overlay + breakpoint genes		 Exporter reads a GFF3 file passed via --annotation file.gff3, embeds gene coords + symbols as genes array in JSON. Scrubber adds (a) a thin gene track on page 1 below the existing tracks, (b) a "genes spanned by candidate" panel on page 2, (c) a "breakpoint-disrupted features" list. Multi-turn — annotation parsing in R, schema extension, three rendering surfaces.
Breakpoint nucleotide context
±50 bp facets, left + right		 Needs the reference FASTA piped into the exporter (--reference ref.fa) so we can grab the actual base sequence around start_bp and end_bp. JSON gets breakpoint_context: {left_seq, right_seq, flanking_bp}. Scrubber renders a sequence panel on page 2. Lighter than IGV but still real work.
IGV.js integration
embedded genome browser		 Embed igv.js (~250 KB), point it at the reference FASTA + per-sample BAM files. Hard part: BAM URLs must be CORS-accessible from the browser. On LANTA, this means standing up an HTTP server with CORS headers (or copying BAMs to a public bucket). Useful when ready, but the data-hosting prerequisite makes this multi-day, not multi-turn.
Manuscript bundle export [done v3.18]
Markdown + TSV + JSON		 Page 4 "📝 manuscript bundle" button. Emits a Markdown file (LLM prompt + candidate tables + per-candidate detail + BibTeX + example paragraph), a TSV (raw rows), and the JSON candidate list. Paste the Markdown into Claude / ChatGPT to draft Methods + Results; the LLM cites with the provided BibTeX keys.
Sidebar collapse + reset layout [done v3.19]
‹ arrow toggle, ↺ reset		 Arrow on the right edge of the sidebar collapses it to a thin strip; click again to expand. State persists across reloads (localStorage). The "↺ reset layout" button in the header restores defaults: sidebar expanded + sim panel at default height (520 px).
Schema v2 layer awareness [done v3.20]
_layers_present + registry export		 Scrubber now detects schema version (v1 inferred / v2 explicit) and tracks state.layersPresent as the source of truth for conditional UI. Schema badge in header shows version + loaded layer count. Page 4 has "📋 export registry (TSV+JSON)" — the canonical contract output for cluster phases 6+ — and "+ load enrichment" to drag-drop layer files from cluster phases. See docs/SCHEMA_V2.md for the full layer contract.
Resize handles between panels
drag to resize each interface		 Currently only the sim panel has a draggable height handle (existing). Adding handles between every interface element (sidebar/main divider, between sim/Z/track panels, between PCA/L3 panels) means: (a) consistent split-pane logic, (b) min/max size enforcement, (c) per-handle persistence in localStorage, (d) graceful redraw on every drag end. Can build but it's a full-turn feature on its own to avoid jank. Until then: sidebar collapse is the easy escape valve, sim panel resize remains as before.
