### 2.1 Candidate registry bridge (priority 1 from turn 128 handoff)

Diagnosed turn 128d: `state.candidateList` (array, mutated by every UI
surface) was disconnected from `state.candidates` (dict, only ever
populated by Save-Session import). The inheritance compute reads from
the dict → empty → I·g pills + grouping panel never lit up.

Fix landed: `_rebuildCandidateRegistries()` derives both dicts from the
arrays. Hooked into:
- `persistCandidateList()` — covers add/remove/JSON-import/list-clear
  (the canonical chokepoint, ~5 mutation entry points)
- `loadCandidateList()` — F5 / cold-start path
- `commitL3Draft()` — also fixed a missing `persistCandidateList()` call
  here (drafts only survived F5 by accident before)
- Save-Session import handler now folds `payload.candidates` into
  `state.candidateList` before the bridge runs, so loaded sessions
  don't get wiped on the next mutation

The bridge **replaces** (not merges) — removed candidates drop from the
dict on the next mutation. Idempotent. Returns `{default_n, detailed_n}`.

Tests: `tests/test_turn129_candidate_registry_bridge.js` (38). Includes
the explicit "pre-bridge gather returns empty / post-bridge returns 3"
proof of the bug-fix.

### 2.3 Representative-near-centroid circles (Quentin-flagged turn 129)

Quentin's screenshot of an R figure with "Circled = representative
individual nearest each group center". Implemented as a small additive
pass in `drawMiniPCA`:
- For each band g0..gK-1 with samples in the pane: compute centroid in
  (PC1, PC2), find argmin Euclidean distance, draw a thin black ring
  (4 px radius, 1.2 px stroke, `themeColor('ink')`)
- Always-on when `labels` exist AND `state.colorMode === 'cluster'`
- Drawn between non-tracked and tracked passes (so a tracked-sample-
  that-is-also-a-rep gets BOTH rings)
- K is derived from `max(labels) + 1` — no hard-coded K
- Composes meaningfully with the het-coloring overlay: rep ring shows
  which dot's het value is most diagnostic of the band

Tests: `tests/test_turn129_l3_representative_circle.js` (10).
