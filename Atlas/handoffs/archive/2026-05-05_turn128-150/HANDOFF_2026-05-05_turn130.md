### 2.2 Slice 1 of fish-trajectory — SHIPPED

Compute pipeline (no UI yet — that's Slice 2). +323 lines in atlas. Six
new functions exposed on `window`:

- `_hungarianChainProjection(l2_indices, K, getLabelsForL2)` — walks L2s
  in order, projects each into the running Hungarian-aligned reference
  frame via the existing `alignLabels()`. Breaks the chain when concord
  drops below `_LINEAGE_CHAIN_BREAK_AGREEMENT` (0.5 default) and starts
  a fresh chain.
- `_concordanceMatrix(projection)` — returns `Float32Array[n*n]` of
  pairwise fish concordance: fraction of L2s where Hungarian-projected
  band of fish i equals fish j. -1 no-call labels excluded from valid
  count.
- `_lineageClustering(concordanceMatrix, n_samples, threshold)` — feeds
  `1 - concordance` into the existing `_agglomerativeAverageLinkage` +
  `_cutDendrogram` (used by the candidate-axis inheritance). Returns
  `lineage_id_per_sample` + `n_lineages`.
- `_lineageCacheKey(l2_indices, K, threshold, mode, chrom)` — stable
  cache key.
- `runLineageCompute(opts)` — orchestrator, reads `state.data`, writes
  `state.lineageResult` + `state.lineageCacheKey`. Bails when
  fewer than `_LINEAGE_MIN_L2_FOR_COMPUTE` (3) L2s.
- `invalidateLineageCache()` — clears the slot.

Result schema:

```js
state.lineageResult = {
  n_samples, n_L2, n_chains,
  threshold, K, mode, chrom,
  chains: [{ l2_indices, n_L2, start_idx }, ...],
  concordance_matrix: Float32Array[n*n],   // square, symmetric, diagonal=1
  dendrogram: [{ left, right, dist, size, members }, ...],
  lineage_id_per_sample: Int32Array[n_samples],
  n_lineages: int,
  fish_count_per_lineage: Int32Array[n_lineages],
}
```

This is the **orthogonal axis** to the existing `inheritanceGroupClustering`:
- Existing: clusters (cand, band) pairs by their fish-membership masks
  ("which bands across candidates carry the same fish?")
- New: clusters fish by their L2-band trajectories
  ("which fish stay together in the same band across the chrom?")

Both run; both will surface in the UI; neither replaces the other.

Tests: `tests/test_turn130_lineage_compute.js` (65). Includes Hungarian
chain projection (identity / aligned / chain-break), concordance matrix
(all-same / two-lineage / no-call exclusion), lineage clustering
(planted recovery / all-identical / strict-cut), runLineageCompute
end-to-end with cache hit, and cache-key composition.
