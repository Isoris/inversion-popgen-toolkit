# MODULE 2A — SNP Discovery & Genotype Likelihood Estimation

## Methods

### Callable-site masking and site frequency spectrum (A01)

Callable genomic regions were defined by extracting intervals of normal ACGT sequence from the haplotype-resolved *C. gariepinus* reference (fClaHyb_Gar_LG, 963.9 Mb across 28 pseudochromosomes) using a custom Python script (`mask_regions_from_fasta.py`). Regions containing runs of ambiguous bases (N), soft-masked repeats (lowercase), or hard-masked sequence were excluded. The callable BED file was converted to ANGSD sites format (1-based coordinates) and indexed with `angsd sites index`.

Per-chromosome region-fold (RF) chunks were constructed for SLURM array parallelization. Site allele frequency (SAF) likelihoods were computed per chunk using ANGSD (`-doSaf 1`, `-anc` set to reference for folded spectrum, `-GL 1`) with shared filters: minimum base quality 25, minimum mapping quality 25, BAQ recalibration (`-baq 1`), excessive mismatch downweight (`-C 50`), individual depth bounds 3–57×, and a minimum of 200 individuals with data at each site (200/226 = 88%). Per-chunk SAF files were merged with `realSFS cat` and the global folded SFS was estimated with `realSFS -fold 1 -bootstrap 100`. The folded SFS was used to compute a per-site allele frequency prior (pest) for downstream SNP calling.

### Biallelic SNP discovery (A02)

Biallelic SNPs were called per chromosome chunk using ANGSD with the pest prior (`-pest`), genotype likelihood model GL=1, and the shared filter set described above. SNP significance was assessed with a likelihood ratio test at P < 1 × 10⁻⁶ (`-SNP_pval 1e-6`) and a minimum minor allele frequency of 0.05 (`-minMaf 0.05`). Only biallelic sites passing all filters were retained (`-doMajorMinor 1 -doMaf 1`).

### Panel construction and distance thinning (A03)

Per-chunk MAF files were merged into a single genome-wide site list. Six distance-thinned panels were generated by retaining only sites separated by at least 200, 500, 1,000, 5,000, 10,000, or 25,000 bp. Fine-thinned panels (200/500/1,000 bp) preserved both major and minor allele columns for downstream BEAGLE generation with locked allele coding (`-doMajorMinor 3`). All panels were indexed with `angsd sites index`.

### Genotype likelihood estimation (A04–A05)

BEAGLE-format genotype likelihood (GL) files were generated per RF chunk and per whole-genome panel using ANGSD (`-doGlf 2`, `-doMajorMinor 3` with locked allele coding from the thinned site files). Per-RF BEAGLE files were merged into whole-genome BEAGLEs, maintaining column (sample) order consistency validated against the BAM list.

### Population structure inference (A06)

Principal component analysis was performed with PCAngsd using 6 eigenvectors (`-e 6`), minimum allele frequency 0.05 (`--minMaf 0.05`), and 250 EM iterations. Ancestry proportions were estimated with NGSadmix for K = 2 through K = 12 using 3 independent random seeds per K value, with minimum allele frequency 0.05. Model fit was evaluated with evalAdmix, which computes pairwise residual correlation matrices between observed and expected genotype likelihoods for each K × seed combination.

The optimal seed per K was selected by a hierarchical decision rule: (1) highest log-likelihood, (2) lowest mean absolute off-diagonal evalAdmix residual, (3) lowest maximum absolute residual, (4) lowest seed number as deterministic tie-break. A stable 12-color palette was assigned to ancestry clusters, and per-sample dominant cluster assignments with full Q-vectors were exported as the canonical ancestry reference table (`sample_main_ancestry_by_K.tsv`).

### Relatedness estimation and kinship pruning (A07)

Pairwise relatedness was estimated from the thin-500 BEAGLE panel using ngsRelate with frequency-based priors. Relatedness classes were assigned using NAToRA (Network Algorithm To Relatedness Analysis) with KING-based theta cutoffs: duplicate/MZ twin (θ > 0.354), first-degree (θ > 0.177), second-degree (θ > 0.0884), and third-degree (θ > 0.0442). A greedy pruning algorithm removed the minimum number of samples to eliminate all first-degree pairs, preferring to retain samples with higher sequencing quality and lower graph degree. The pruned sample list was used for a second round of structure inference to confirm that ancestry patterns were robust to kinship structure.

### Combined structure summaries (A08)

Best-seed-by-K tables from the full (226-sample) and pruned (unrelated) analyses were merged into a single combined table with a `sample_set` column (all | pruned), providing a unified reference for downstream modules.
